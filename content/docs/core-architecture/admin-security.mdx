# Админка, UI и Безопасность

В этом разделе описывается, как мы делаем панель управления WordPress чистой, понятной и безопасной для Enterprise B2B проектов. Вся описанная логика находится в директориях `admin/` и `security/` внутри `mu-plugins/agency-b2b-core/`.

---

## 1. Интерфейс Админки (Admin UI)

Скрипт `security/admin-ui.php` очищает интерфейс от "мусора", чтобы контент-менеджер не отвлекался.

### 1.1 Очистка Дашборда (Главного экрана)
По умолчанию WordPress выводит множество лишних виджетов ("События и новости", "Активность"). Мы их безжалостно удаляем.
Вместо мусора мы выводим один кастомный **виджет-помощник** (`agency_welcome_widget`), где крупными кнопками предлагаем менеджеру быстрые действия (например, "Добавить услугу" или "Обратиться в техподдержку").

### 1.2 Брендирование страницы входа
Мы полностью брендируем страницу `/wp-login.php`:
* Через хук `login_enqueue_scripts` загружается CSS, который заменяет дефолтный логотип WordPress на логотип клиентского проекта.
* Кнопка "Войти" перекрашивается в корпоративные цвета клиента.
* Хук `login_headerurl` направляет клик по логотипу на главную страницу сайта, а не на `wordpress.org`.

### 1.3 Скрытие лишнего (RankMath)
SEO-плагины часто перегружают интерфейс. Для редакторов (пользователей, не являющихся администраторами) мы скрываем панель RankMath в Gutenberg через CSS, чтобы они случайно не сломали SEO-настройки.

## 2. Инструменты: Импорт и Экспорт Каталога

Для сложных B2B каталогов необходима выгрузка и загрузка через CSV (например, для связки с МойСклад). Логика реализована в `admin/import-export-page.php`. Прямо в админке, в меню Каталога появляется подраздел **Импорт / Экспорт**.

<Steps>
  <Step title="Экспорт">
    Пользователь нажимает кнопку выгрузки. Форма отправляет POST запрос (`action = agency_catalog_export`). Хук "на лету" собирает данные из постов `catalog_product` и мета-полей Carbon Fields, отдавая пользователю готовый CSV-файл.
  </Step>
  <Step title="Импорт: Загрузка и Маппинг">
    Менеджер загружает CSV. Скрипт сохраняет его во временную папку `uploads/agency-import/`. Считывается первая строка CSV и выводится интерфейс, где пользователь сопоставляет колонки из Excel (например, "Артикул 1С") с внутренними полями базы данных (например, `catalog_sku`).
  </Step>
  <Step title="Импорт: AJAX обработка">
    Запускается пошаговый JavaScript (`import-export.js`), который пачками (по 50 товаров) отправляет данные на сервер. Это гарантирует, что сервер не упадет с ошибкой `504 Gateway Timeout` при обработке десятков тысяч товаров.
  </Step>
</Steps>

## 3. Безопасность ролей и Контента

Защита "от дурака" (и от хакеров) встроена в ядро на трех уровнях.

<Cards>
  <Card title="Защита SVG (Sanitizer)">
    **Файл:** `security/svg-sanitizer.php`
    
    WordPress запрещает загрузку SVG, так как в него можно встроить XSS-скрипт. Мы разрешаем загрузку, но жестко прогоняем файлы через санитайзер при загрузке (на хуке `wp_handle_upload_prefilter`). Контент-менеджеры могут загружать векторные иконки, не подвергая сайт риску.
  </Card>
  <Card title="Ограничение ролей (Редакторы)">
    **Файл:** `security/roles-caps.php`
    
    Менеджерам дается роль **Editor (Редактор)**, но ее права сильно урезаны:
    * Запрещено удалять опубликованные базовые страницы.
    * Полностью закрыт доступ к плагинам, темам и любым настройкам.
  </Card>
  <Card title="Блокировка Gutenberg (Template Lock)">
    **Файл:** `security/gutenberg-lock.php`
    
    В тех CPT, где разрешен Gutenberg (например, сложный лендинг), мы можем применить `template_lock = 'all'`. Это означает, что контент-менеджер может **изменять текст и картинки** внутри существующих блоков, но не может удалять сами блоки или добавлять новые. БЭМ-верстка остается нерушимой.
  </Card>
</Cards>
