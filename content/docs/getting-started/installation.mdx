# Быстрый старт: Установка и Запуск

В этом руководстве мы пройдем весь путь от создания нового репозитория до первого запуска сайта с использованием встроенного Мастера Установки (Agency Setup).

---

## 1. Git: Клонирование и Upstream

Главное правило нашего бойлерплейта — **мы не теряем связь с материнским репозиторием (ядром)**. Это позволяет в будущем получать обновления безопасности и новые фичи без боли.

<Steps>
  <Step title="Клонирование Бойлерплейта">
    Скачайте Boilerplate на локальную машину:
    ```bash
    git clone git@github.com:your-agency/agency-b2b-boilerplate.git my-client-project
    cd my-client-project
    ```
  </Step>
  <Step title="Связка с клиентом (Origin) и Ядром (Upstream)">
    Переименуйте оригинальный удаленный репозиторий в `upstream` (это будет наше Ядро), и добавьте новый репозиторий клиента как `origin`:
    ```bash
    git remote rename origin upstream
    git remote add origin git@github.com:your-agency/my-client-project.git
    git push -u origin main
    ```
  </Step>
  <Step title="Получение обновлений в будущем (Только Ядро!)">
    Спустя полгода, когда в Ядре выйдет важный багфикс или фича, мы вытягиваем **только** обновления бэкенда (`agency-b2b-core`). Мы **не делаем полный merge**, так как тема (`agency-starter-theme`) уже кастомизирована под клиента!
    
    ```bash
    git fetch upstream
    
    # Вытягиваем изменения ТОЛЬКО для папки ядра (путь может отличаться в зависимости от структуры)
    git checkout upstream/main -- wp-content/mu-plugins/agency-b2b-core/
    
    git commit -m "chore: Обновление ядра из Boilerplate"
    ```
    *(Также можно использовать `git cherry-pick` для выборочного переноса конкретных коммитов).*
  </Step>
</Steps>

## 2. Локальная разработка (Docker)

Для запуска локального сервера мы используем Docker, чтобы у всей команды было одинаковое окружение (PHP 8+, MySQL).

<Callout type="info">
  **Рекомендуется:** Если вы работаете на Windows (WSL2) или Mac, используйте конфигурацию `light`. Она не монтирует ядро WordPress целиком, что избавляет от проблем с медленной файловой системой (I/O).
</Callout>

Запустите контейнеры одной командой в корне проекта:
```bash
docker-compose -f docker-compose.light.yml up -d
```
Сайт будет доступен по адресу `http://localhost:8080`.

## 3. Мастер Установки (Agency Setup)

Мы автоматизировали скучную рутину инициализации WordPress. Зайдите в админ-панель WordPress (`/wp-admin`) под логином администратора. 

В левом меню вы увидите раздел **Agency Setup** (Мастер установки).

Вам нужно просто заполнить базовые данные (например, Email для заявок `shaynyrov22@mail.ru`) и прокликать 4 кнопки по порядку:

<Steps>
  <Step title="1. Активировать плагины и применить настройки">
    Скрипт автоматически активирует Carbon Fields, SVG Sanitize и другие Must-Use плагины, а также настроит постоянные ссылки (ЧПУ) и отключит лишние виджеты.
  </Step>
  <Step title="2. Создать структуру базовых страниц">
    Будут сгенерированы системные страницы: "Главная", "Контакты", "Политика конфиденциальности" и "404". Страница "Главная" автоматически будет назначена стартовой в настройках чтения.
  </Step>
  <Step title="3. Сгенерировать формы Contact Form 7">
    Бойлерплейт создаст готовую форму обратной связи с правильными B2B-полями (Имя, Телефон, Компания) и настроит отправку писем на указанный вами Email.
  </Step>
  <Step title="4. Сгенерировать базовые SEO-теги">
    *Внимание: перед нажатием убедитесь, что заполнили массив данных в файле `scripts/seed-seo.php`.*
    Скрипт пропишет Title, Description и OpenGraph-теги для всех базовых страниц.
  </Step>
</Steps>

## 4. Активация темы

Последний штрих:
1. Перейдите в **Внешний вид -> Темы**.
2. Активируйте тему **Agency Starter Theme**.
3. Нажмите "Перейти на сайт". 

**Поздравляем! Ваш B2B-проект готов к разработке бизнес-логики и верстке.**
