# Обновления и Регламенты

Когда Boilerplate (`upstream`) получает новые фичи или фиксы безопасности, нам нужно доставить их на клиентские сайты (`origin`).

## Процесс обновления клиента (без конфликтов)

Прошел год, мы добавили в ядро бойлерплейта новый модуль "Отзывы". Чтобы обновить клиента:

1. Зайдите в проект клиента.
2. Выполните:
   ```bash
   git fetch upstream
   git merge upstream/main
   ```
3. Так как вы **не удаляли** и **не меняли** файлы в `mu-plugins/agency-b2b-core/` напрямую у клиента, Git проведет слияние без единого конфликта!

<Callout type="info">
  Новая фича "Отзывы" прилетит клиенту, но она по умолчанию скрыта за Feature Toggle. Сайт продолжит работать как раньше. Если клиент захочет отзывы, просто добавьте `add_theme_support('agency-cpt-reviews')`.
</Callout>

## Отказ от обновлений UI

Как только вы начали верстать уникальный дизайн для клиента, тема `agency-starter-theme` на его сайте больше не нуждается в обновлениях "универсальных" стилей из бойлерплейта. 

Вытягивайте из `upstream` (через `git cherry-pick` или `merge` конкретных папок) только обновления бэкенда (`agency-b2b-core`), но не трогайте уникальную CSS верстку клиента.

## Чек-лист перед коммитом

Перед каждым push в `origin` или `upstream` проверяйте себя:
- [ ] **Безопасность**: Все данные на выводе пропущены через `esc_html()`, формы имеют `wp_nonce_field()`.
- [ ] **PHP 8.1+**: Используется строгая типизация `declare(strict_types=1);` и неймспейсы.
- [ ] **Никакого ACF**: Все поля написаны на Carbon Fields.
- [ ] **Никакого хардкода в CSS**: Везде используются переменные (tokens).
