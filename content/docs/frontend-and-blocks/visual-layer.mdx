# Визуальный слой (Тема, Токены, БЭМ)

Этот раздел описывает то, как мы пишем CSS, почему мы не используем Tailwind или Bootstrap в Boilerplate, и как работает производительная загрузка стилей в нашей теме `agency-starter-theme`.

---

## 1. Архитектура CSS: Токены и БЭМ

Мы используем чистый CSS с методологией БЭМ (Блок-Элемент-Модификатор). В файле `style.css` заложена строгая дизайн-система из трех уровней токенов (CSS переменных).

### Почему 3 уровня токенов?
Это стандарт Enterprise-дизайна, который делает тему неубиваемой при глобальном редизайне.

<Steps>
  <Step title="Primitives (Примитивы)">
    Это палитра. Базовые цвета и размеры. Мы *почти никогда* не используем их напрямую в стилях компонентов.
    ```css
    --color-primary: #6b7280;
    --spacing-md: 16px;
    ```
  </Step>
  <Step title="Semantics (Смысловые роли)">
    Это функциональные роли, которые ссылаются на примитивы. Именно их мы используем в компонентах.
    ```css
    --color-bg-interactive: var(--color-primary);
    --radius-interactive: var(--radius-sm);
    ```
    *Зачем это?* Если завтра клиент скажет: "Хочу круглые кнопки, а карточки квадратные", мы меняем один токен `--radius-interactive: 50px`, и это не ломает радиус карточек (`--radius-card`)!
  </Step>
  <Step title="Global Layout">
    Настройки глобальной сетки (например, максимальная ширина контейнера `1280px`, отступы секций).
  </Step>
</Steps>

### Избегаем жестких модификаторов (Inline Tokens)
Вместо того чтобы плодить классы вроде `.btn--round`, `.btn--red`, мы используем **инлайн-токены** поверх дефолтных fallback-значений.

**CSS (Файл компонента):**
```css
.btn {
    /* Если переменная --btn-radius задана, используем её, иначе фоллбэк */
    border-radius: var(--btn-radius, var(--radius-interactive));
    background-color: var(--btn-bg, var(--color-bg-interactive));
}
```

**HTML (или настройки блока Gutenberg):**
```html
<!-- Круглая красная кнопка без создания новых CSS-классов! -->
<button class="btn" style="--btn-radius: 50px; --btn-bg: #ff0000;">Нажать</button>
```

## 2. Темная тема (Dark Mode)

Благодаря семантическим токенам, темная тема реализуется в **10 строк кода**. Мы просто переопределяем базовые "Примитивы" для атрибута `data-theme="dark"`:

```css
:root[data-theme="dark"] {
    --color-primary: #9ca3af;
    --color-surface: #1f2937;
    --color-background: #111827;
    --color-text-main: #f9fafb;
}
```
Весь сайт перекрашивается автоматически. Никаких уродливых конструкций типа `.dark-mode .card { background: #333; }` писать не нужно!

<Callout type="info">
  **Борьба с морганием (FOUC):**
  Переключение происходит в JS. А чтобы сайт не "моргал" белым при загрузке темной темы, в `header.php` вставлен синхронный блокирующий скрипт, который мгновенно читает `localStorage` до рендера DOM.
</Callout>

## 3. Модульная загрузка стилей (Performance)

Мы не собираем все стили в один гигантский `style.min.css`. В `functions.php` используется условная загрузка ("on-demand"). Стили компонентов грузятся **только там, где они нужны**:

```php
// Подключается ВСЕГДА (Header, Footer, Кнопки)
wp_enqueue_style( 'agency-header', get_template_directory_uri() . '/assets/css/components/header.css', [], _S_VERSION );

// Подключается ТОЛЬКО в каталоге
if ( is_search() || is_post_type_archive( 'catalog_product' ) || is_tax( 'catalog_category' ) ) {
    wp_enqueue_style( 'agency-archive-catalog', get_template_directory_uri() . '/assets/css/templates/archive-catalog.css', ['agency-catalog-grid'], _S_VERSION );
}
```
*Зачем это?* Идеальная оценка Google PageSpeed. Браузеру не нужно парсить стили и скрипты корзины или карточки товара, если пользователь просто читает текстовую статью в блоге.

## 4. Компонентный подход в PHP (`template-parts`)

Архитектура темы построена на строгом БЭМ и изолированных компонентах (в папке `template-parts/`). Это позволяет **свободно переключать части сайта**.

### Пример: Подключение шапки (Header)
В файле `header.php` нет жесткой верстки шапки. Там только HTML-каркас и вызов:
```php
<?php get_template_part('template-parts/header/layout', 'minimal'); ?>
```
*   Если для B2B клиента нужна строгая шапка, вы используете `layout-minimal.php`. 
*   Если нужен интернет-магазин с корзиной, вы создаете `layout-ecommerce.php` и просто меняете вызов в `header.php`.

### Пример: Вывод Архивов
В шаблоне архива (`archive-catalog_product.php`) мы не пишем HTML карточки. Внутри цикла `while (have_posts())` мы вызываем:
```php
get_template_part('template-parts/catalog/card');
```
Это гарантирует, что если мы изменим дизайн карточки товара (добавим кнопку "В избранное" в `card.php`), она **мгновенно обновится везде**: в основном каталоге, в блоке "Похожие товары" и в результатах поиска.
