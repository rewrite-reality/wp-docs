# Frontend и визуальный слой

Визуальный слой (`agency-starter-theme`) отвечает только за рендеринг. Здесь нет бизнес-логики.

## CSS Архитектура и Токены

Мы верстаем строго по **БЭМ** и используем трехуровневую систему токенов (CSS переменных). Хардкод в пикселях строго запрещен!

<Accordions type="single">
  <Accordion title="1. Primitives (Базовые шкалы)">
    Базовые цвета и размеры, которые не используются в верстке напрямую.
    `--gray-100: #f5f5f5;`
    `--space-4: 1rem;`
  </Accordion>
  <Accordion title="2. Semantics (Смысловые роли)">
    Переменные, которые наследуются от примитивов и отвечают за смысл элемента.
    `--radius-interactive: 8px;`
    `--bg-primary: var(--gray-100);`
  </Accordion>
  <Accordion title="3. Global Layout (Отступы)">
    Переменные для глобальной сетки.
    `--section-padding-desktop: 120px;`
  </Accordion>
</Accordions>

### Семантика и Fallbacks

При стилизации компонентов в БЭМ-блоках используйте фоллбэки. Запрещено создание жестких визуальных классов (типа `.btn--round`).

```css
.button {
    /* Если переменная блока не задана, берется глобальный семантический токен */
    border-radius: var(--btn-radius, var(--radius-interactive));
}
```

## Работа с Gutenberg Блоками

Gutenberg используется как конструктор из готовых атомарных блоков. 

<Callout type="info">
  Стили для блоков должны загружаться изолированно только на тех страницах, где блок реально используется.
</Callout>

Для этого в WordPress используйте:
```php
wp_enqueue_block_style('agency/custom-block', [
    'handle' => 'agency-custom-block',
    'src'    => get_theme_file_uri('/assets/css/blocks/custom-block.css'),
]);
```

## JavaScript

**Никакого jQuery на фронтенде!** Используйте только Vanilla JS (ES6+ классы).
