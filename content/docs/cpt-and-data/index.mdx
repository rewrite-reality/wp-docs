# Гибридная модель CPT

В нашем бойлерплейте используется **гибридная архитектура данных**, разделяющая типы постов на два вида.

## 1. Narrative CPT (Описательные)
К ним относятся сущности вроде **Услуги** или **Страницы**, где контент уникален, неструктурирован и требует свободы верстки.

* **Инструмент:** Gutenberg (атомарные блоки).
* **Особенности:** Используются Block Templates, мета-поля сведены к минимуму (только SEO или базовые настройки).

## 2. Structured CPT (Структурированные)
К ним относятся **Кейсы** или **Товары**, где дизайн жестко задан, а контент представляет собой набор характеристик (цена, год, результат).

* **Инструмент:** Carbon Fields (никакого ACF!).
* **Особенности:** Gutenberg жестко отключен. Вывод осуществляется через классические PHP-шаблоны темы.

<Callout type="warning">
  Мы не используем ACF. Все кастомные поля регистрируются исключительно через код с помощью **Carbon Fields**.
</Callout>

## Изменение полей без удаления ядра

Если клиенту нужно убрать поле (например, "Бюджет" у Кейса), **не удаляйте** его из `agency-b2b-core`. Используйте фильтры в клиентском плагине `client-core.php`:

```php
// В mu-plugins/client-core.php:
add_filter('agency_cases_meta_fields', function($fields) {
    // Отфильтровываем поле по имени
    return array_filter($fields, fn($f) => $f->get_name() !== 'budget');
});
```
